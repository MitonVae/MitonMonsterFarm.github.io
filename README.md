# 怪兽农场游戏

这是一个怪兽农场经营模拟游戏，玩家可以培育作物、捕获怪兽、探索世界并发展自己的农场。项目还包含一个用于自动化测试游戏数值平衡的模拟系统。

## 项目结构

```
.
├── game/              # 游戏主文件
│   ├── assets/        # 图片和其他资源
│   ├── css/           # 样式文件
│   ├── js/            # 游戏脚本
│   ├── index.html     # 游戏主页面
│   └── start_python_server.bat # 启动本地服务器脚本
│
├── tools/             # 工具目录
│   └── simulator/     # 平衡性模拟工具
│       ├── balance_sim.js    # 核心模拟器类
│       ├── run_sim.js        # 运行器
│       └── sim_config.js     # 配置文件
│
└── docs/              # 游戏文档和Wiki
```

## 游戏特性

1. **全面的游戏机制模拟**
   - 作物生长与收获系统
   - 怪兽捕获与维护系统
   - 资源产出与消耗平衡
   - 游戏进程和阶段推进

2. **多种策略模拟**
   - 最优策略 (optimal): 最高效率利用资源
   - 平衡策略 (balanced): 均衡发展各方面
   - 资源积累策略 (resource-focused): 优先积累资源再发展
   - 科技专注策略 (tech-focused): 优先解锁关键科技
   - 繁殖专注策略 (breeding-focused): 优先发展怪兽数量和质量

3. **详细的分析报告**
   - 资源产出/消耗曲线
   - 游戏进程速度评估
   - 怪兽维护成本效益分析
   - 平衡性评分与建议

## 使用方法

### 基础运行

```bash
node run_sim.js
```

这将执行预设的多组测试方案，并在控制台输出简要结果，同时在 `balance_reports` 目录下生成详细的JSON报告文件。

### 自定义测试

1. 修改 `sim_config.js` 中的 `presetTests` 部分，添加或调整测试配置
2. 也可以直接修改 `run_sim.js` 中的 `tests` 数组，自定义测试方案

### 配置参数说明

#### 模拟器配置选项

- **simSpeed**: 模拟速度倍率，值越大模拟越快
- **logInterval**: 日志记录间隔（秒）
- **endTime**: 模拟结束时间（秒）
- **strategyType**: 模拟策略类型

#### 平衡参数覆盖

在 `balanceOverrides` 对象中可以覆盖默认的平衡参数，例如：

```javascript
{
  "crop.yieldBase": 1.25,    // 作物产量提高25%
  "monsterUpkeep.common.food": 0.05  // 普通怪兽食物消耗降低
}
```

## 平衡性评分说明

系统基于以下几个维度对游戏平衡性进行评分（1-10分）：

1. **资源平衡 (Resource Balance)**
   - 考量各类资源的增长率和波动性
   - 理想状态：所有资源稳定增长，波动较小

2. **游戏进程 (Progression Pace)**
   - 评估从一个游戏阶段到下一阶段所需的时间
   - 理想状态：每个阶段约2-4小时游戏时间

3. **怪兽效益 (Monster Value)**
   - 分析怪兽维护成本与其产出的比值
   - 理想状态：产出显著高于维护成本，但不至于过分失衡

4. **总体平衡 (Overall Balance)**
   - 综合以上三项的加权平均
   - 理想状态：7分或以上表示良好平衡

## 自定义模拟逻辑

如果需要调整模拟逻辑，可以修改以下几个部分：

1. `balance_sim.js` 中的 `BalanceSimulator` 类的相关方法
2. `run_sim.js` 中的 `EnhancedSimulator` 类扩展方法
3. 特别是 `_executeOptimalStrategy` 等策略方法，可以根据需要调整AI行为

## 注意事项

- 这是一个模拟系统，其结果提供参考，但不能完全替代真实玩家测试
- 由于各种随机因素和简化处理，模拟结果会有一定偏差
- 建议将模拟结果与少量玩家测试结合，获取更准确的平衡调优方案
